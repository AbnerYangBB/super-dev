# Web AGENTS 协作规范

## 1) 沟通与交付
1. 注释、分析、回复统一使用中文，技术术语可保留英文。
2. 先说明页面目标、约束和验收标准，再开始实现。
3. 输出必须说明改动文件、验证结果、未执行项及原因。

## 2) Web 实现原则
1. 优先复用现有组件、设计 token 和构建配置，禁止无理由重做基础设施。
2. 涉及页面、组件、交互设计时优先使用 `frontend-design` skill。
3. 兼顾桌面端与移动端，默认包含基础可访问性检查。

## 3) 工程边界
1. 优先修改 AI 配置、前端实现和相关文档，不擅自改动无关后端逻辑。
2. 若需要浏览器自动化或外部文档，优先通过 MCP 或现有技能完成。
